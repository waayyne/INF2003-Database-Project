SQL View: product_summary_view

This view combines product and brand information into one virtual table. It makes repeated product queries easier to write because the join between products and brands is already stored inside the view. 

CREATE VIEW product_summary_view AS
SELECT
    p.product_id,
    p.product_name,
    b.brand_name,
    p.price_usd,
    p.rating,
    p.reviews,
    p.loves_count,
    p.out_of_stock
FROM products p, brands b
WHERE p.brand_id = b.brand_id;

View Query Result

SELECT product_id, product_name, brand_name, price_usd, rating, reviews
FROM product_summary_view
WHERE rating >= 4.5
ORDER BY rating DESC
LIMIT 20;

SQL Indexes on Products Table

Indexes are created on frequently searched, filtered, and joined columns to improve query performance. In this project, indexes help product searches, brand joins, rating filters, and price filters. 

CREATE INDEX idx_products_product_name ON products(product_name);
CREATE INDEX idx_products_brand_id ON products(brand_id);
CREATE INDEX idx_products_rating ON products(rating);
CREATE INDEX idx_products_price ON products(price_usd);

Index Performance: EXPLAIN Comparison

This section compares the same rating query twice. The first query forces MariaDB to ignore the rating index, while the second query allows MariaDB to use the idx_products_rating index normally.

The important column is rows. It shows how many rows MariaDB estimates it needs to scan. A lower number usually means the index is helping.

Without Index: IGNORE INDEX

EXPLAIN SELECT product_id, product_name, rating, price_usd
FROM products IGNORE INDEX (idx_products_rating)
WHERE rating >= 4.5
ORDER BY rating DESC;
