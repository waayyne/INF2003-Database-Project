SQL View: product_summary_view

This view combines product and brand information into one virtual table. It makes repeated product queries easier to write. 

CREATE VIEW product_summary_view AS
SELECT
    p.product_id,
    p.product_name,
    b.brand_name,
    p.price_usd,
    p.rating,
    p.reviews,
    p.loves_count,
    p.out_of_stock
FROM products p, brands b
WHERE p.brand_id = b.brand_id;

SQL Indexes on Products Table

Indexes are created on frequently searched, filtered, and joined columns to improve query performance.

CREATE INDEX idx_products_brand_id ON products(brand_id);
CREATE INDEX idx_products_rating ON products(rating);
CREATE INDEX idx_products_price ON products(price_usd);

Index Performance: EXPLAIN Comparison

The same query is run twice — once forcing a full table scan by ignoring the index, and once using the idx_products_rating index normally. The rows column shows how many rows MariaDB had to scan to get the result. A lower number means the index is working.

Without index (IGNORE INDEX)

EXPLAIN SELECT product_id, product_name, rating, price_usd
FROM products IGNORE INDEX (idx_products_rating)
WHERE rating >= 4.5
ORDER BY rating DESC;
