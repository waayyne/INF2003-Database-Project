SQL View

product_summary_view

This virtual table combines product and brand information for repeated product queries.

products + brands

View Definition

CREATE VIEW product_summary_view AS
SELECT
    p.product_id,
    p.product_name,
    b.brand_name,
    p.price_usd,
    p.rating,
    p.reviews,
    p.loves_count,
    p.out_of_stock
FROM products p, brands b
WHERE p.brand_id = b.brand_id;

Why It Helps

Instead of rewriting the same join repeatedly, the view gives the admin pages a reusable product summary shape.

Reusable query Cleaner reports Less repeated SQL

SQL Indexes

Products Table Indexes

Indexes are created on frequently searched, filtered, and joined columns.

query speed

Index Commands

CREATE INDEX idx_products_product_name ON products(product_name);
CREATE INDEX idx_products_brand_id ON products(brand_id);
CREATE INDEX idx_products_rating ON products(rating);
CREATE INDEX idx_products_price ON products(price_usd);

Where They Matter

  • Product name search
  • Brand joins
  • Rating filters and sorting
  • Price range filters
