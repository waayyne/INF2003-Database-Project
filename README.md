# INF2003 Database Systems — Group Project

## Project Overview

This project focuses on the design and development of a database driven application using both:

- Relational Database Systems
- Non-Relational (NoSQL) Database Systems

The project aims to simulate a real world database application that demonstrates practical database design, implementation, querying, and system integration skills.

Students are encouraged to build an application with:
- meaningful business logic
- real world datasets
- graphical user interface or web application
- scalable database architecture

The project allows students to apply concepts learned throughout the module in a practical and collaborative environment.

---

# Objectives

The main objectives of this project are:

- design a relational database system
- design a NoSQL database system
- compare SQL and NoSQL approaches
- implement CRUD operations
- work with real world datasets
- explore advanced database features
- apply database concepts in practical applications
- develop teamwork and collaboration skills

---

# Project Requirements

The application must include:

- a relational database backend
- a NoSQL database backend
- minimum 3 interconnected tables/entities
- different relationship types
- CRUD operations
- complex database queries
- graphical or web based interface (recommended)

The system should demonstrate:
- data modeling
- database normalization
- NoSQL design concepts
- query implementation
- business workflow logic

## 1. Identify an Application

Students must select a suitable application domain that:

- transportation systems

# Database Design

## Relational Database Requirements

Students must:
- construct an ER diagram
- create at least 3 tables
- implement different relationship types

Relationship examples:
- one-to-many
- many-to-one
- many-to-many

Data types should include:
- strings
- integers
- dates
- numerical values

---

## NoSQL Database Requirements

Students must:
- design a NoSQL data model
- justify database design decisions
- implement suitable NoSQL structures

Possible NoSQL models:
- document databases
- key-value databases
- column databases
- graph databases

Students should discuss:
- advantages and disadvantages
- schema flexibility
- scalability considerations

---
